Basic Server Commands

Once you are logged into the CS server, here are some essential commands to get started:

pwd - Print Working Directory

Shows your current location in the file system:

krishna@pe15:~$ pwd
/home/krishna

ls - List Files and Directories

Displays the files and folders in your current directory:

krishna@pe15:~$ ls
cs110  cs210  cs310  cs410  public_html

whoami - Display Current User

Shows the username you are logged in as:

krishna@pe15:~$ whoami
krishna

cd - Change Directory

Navigate to a different folder:

krishna@pe15:~$ cd cs410
krishna@pe15:~/cs410$

cat - View File Contents

Display the contents of a text file:

krishna@pe15:~/cs410$ cat R18.txt
Krishna Kanodia
R18
...

mkdir - Create a New Directory

Create a new folder in your current location:

krishna@pe15:~$ mkdir myproject
krishna@pe15:~$ ls
cs110  cs210  cs310  cs410  myproject  public_html

Troubleshooting Common SSH Errors

Here are some common errors you might encounter when connecting to the CS server and how to fix them:

Error: "Connection refused"

Problem: The server is not accepting SSH connections or is down for maintenance.

Solution:

  • Check if you typed the server address correctly: users.cs.umb.edu
  • Verify your internet connection is working
  • Wait a few minutes and try again - the server might be temporarily down
  • Contact the CS department if the problem persists

Error: "Permission denied (publickey,password)"

Problem: Your username or password is incorrect.

Solution:

  • Double-check your username - it should be your CS server username, not your UMB email
  • Make sure CAPS LOCK is off when typing your password
  • Try resetting your password through the CS department
  • Verify you're using the correct format: ssh username@users.cs.umb.edu

Error: "Host key verification failed"

Problem: The server's security key has changed or there's a potential security risk.

Solution:

  • This usually happens after server maintenance or upgrades
  • Remove the old key from your known_hosts file:
  • Windows: ssh-keygen -R users.cs.umb.edu
  • Then try connecting again and type "yes" when prompted to accept the new key

Error: "ssh: command not found"

Problem: SSH is not installed on your computer or not in your system path.

Solution:

  • Windows: Use PowerShell (not Command Prompt). SSH is built into Windows 10/11 PowerShell
  • Older Windows: Install PuTTY or Bitvise
  • Mac/Linux: SSH should be pre-installed. Try opening Terminal and typing which ssh

Error: "Connection timed out"

Problem: Your network is blocking the SSH connection or the server is unreachable.

Solution:

  • Check your firewall settings - some networks block SSH (port 22)
  • Try from a different network (home, campus WiFi, mobile hotspot)
  • If on campus, make sure you're connected to the UMB network
  • VPN connections can sometimes interfere - try disconnecting VPN

Still having issues? Contact the CS department help desk or email cs-support@umb.edu for assistance.

Pro Tips: Keyboard Shortcuts & Command History

Master these time-saving shortcuts to work more efficiently on the CS server:

⌨️ Essential Keyboard Shortcuts

Ctrl + C Cancel/stop the current command
Ctrl + D Log out of the SSH session (same as typing exit)
Ctrl + L Clear the terminal screen (same as typing clear)
Ctrl + A Move cursor to the beginning of the line
Ctrl + E Move cursor to the end of the line
Ctrl + U Delete everything before the cursor
Ctrl + K Delete everything after the cursor
Tab Auto-complete file/directory names and commands

🕐 Command History Navigation

Arrow Keys:

  • ↑ (Up Arrow) - View previous command in history
  • ↓ (Down Arrow) - View next command in history

History Command:

krishna@pe15:~$ history
  498  cd cs410
  499  ls
  500  cat R18.txt
  501  pwd
  502  history

Useful History Tricks:

  • !! - Run the last command again
  • !500 - Run command number 500 from history
  • !cat - Run the most recent command starting with "cat"
  • Ctrl + R - Search through command history (type to search, Enter to run)

💡 Bonus Tips

  • Use Tab completion: Type the first few letters of a file/directory name and press Tab to auto-complete. If multiple matches exist, press Tab twice to see all options.
  • Copy/Paste in terminal: Right-click to paste in PowerShell. Some terminals use Ctrl+Shift+C and Ctrl+Shift+V instead of Ctrl+C/V.
  • Clear screen quickly: Instead of typing clear, just press Ctrl + L.
  • Exit without typing: Press Ctrl + D to log out instead of typing exit.
  • Fix typos fast: Use Ctrl + A to jump to the start of a long command to fix a mistake at the beginning.
